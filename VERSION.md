# ShadowHunters 版本紀錄（使用者版）

## v0.2.3-test (2026-04-04)

### 修正項目

#### Issue 62 - 重啟後房間恢復與閒置房自動清理
- 服務重新啟動後，招募中與進行中的房間現在會自動恢復，不會再整批消失。
- 房內原本的玩家名單、聊天紀錄與進行中的對局狀態會一併延續，玩家中斷感降低。
- 若房間超過 1 小時都沒有任何新的聊天或系統提示，系統會自動廢除該房間，避免列表長期堆積過期村莊。
- 已超過閒置時間的舊房間在服務重啟後也不會再被重新掛回列表。
- 若伺服器維修停機一段時間後才重新開啟，房間閒置倒數與回合暴斃倒數都會把停機期間一起算進去，會從原本剩餘時間接續，不會因重開而重置。

#### Issue 63 - 中立大亂鬥歷史紀錄標記補正
- 已修正部分歷史對局雖然實際開了中立大亂鬥，但回放頁與紀錄列表沒有顯示標記的問題。
- 現在像 2336 村、2340 村這類已結束村莊，查看紀錄時也會正確顯示中立大亂鬥選項徽章。
- 系統會自動補齊舊紀錄缺漏的設定資訊，之後查閱其他同類型歷史村莊時也能正常辨識。
- 中立大亂鬥的 `C` 選項徽章底色已調整為與中立陣營一致的黃色，辨識更直覺；房間內卡組摘要也不再重複顯示 `[C]`。

#### Issue 58 - 丹尼爾在無暗影對局的勝利判定修正

#### Issue 60 - 場地卡說明文字閱讀性調整

- 場地標題與數字辨識色仍保留，不影響原本的場地識別感。

#### Issue 61 - Windows 啟動時額外跳出 Python 視窗修正
- 在 Windows 重新啟動本機服務時，自動建立測試房的背景程序已改為安靜執行，不會再額外多跳一個 Python 視窗干擾操作。
- 既有自動建立測試房功能仍會保留；若不需要，也可在啟動設定中關閉。
